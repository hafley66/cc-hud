All tasks from the original plan are complete. Several tooltip positioning issues were discovered and fixed during the session.

## Problem/Context
Legend panel had no aggregate stats view. Budget chart used egui_plot's built-in label_formatter which just showed the Y value inline on the chart. Tooltip positioning caused multiple issues: covering chart bars, covering the budget chart when hovering it, and VLine coordinate mismatch causing violent flashing in non-time mode.

## Solution/Approach
- Stats strip: 32px fixed allocation inside panel_frame before ScrollArea. Items evenly distributed with draw_spread helper.
- Budget tooltip: Added HoverSource::Budget variant, wired update_hover_src + try_pin into budget chart show() closure, added budget header to running_total_header with date/spent/remaining/rate/projected.
- VLine fix: In non-time mode, skip VLine when prev_hover source and current chart use different coordinate systems (budget=time-x, others=turn-x).
- Budget tooltip position: right edge of tooltip aligns to left edge of usage_rect so it never covers the budget chart.
- Tooltip Y overflow: falls below cursor (+20px) when it cannot fit above.

- `LegendStats` struct extended with `agent_cost: f64`, four construction sites updated
- `SessionData.first_ts`/`last_ts` used for active hours computation

- Stats strip items distributed via `draw_spread` closure: divides strip width into N equal slots, centers each item in its slot
