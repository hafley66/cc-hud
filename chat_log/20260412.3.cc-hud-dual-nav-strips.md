# Session: cc-hud-dual-nav-strips

## Goal
Fix confused nav/view state. Time mode = 1 nav bar; turn mode = 2 bars (time filters sessions by last_ts window, turn zooms chart x-axis 0..max_turns).

## Current State
- Committed 8876fd1.
- draw_nav_strip helper extracted.
- Layout: nav_rect always; turn_nav_rect stacked below in turn mode.
- max_turns computed from ALL sessions (unfiltered) so turn domain doesn't shrink when time strip narrows.
- Time nav always drives time_view. Turn nav drives turn_view. active_view (for charts) picks turn_view in non-time, time_view in time.
- turn_view default: first 200 turns on init.
- Autofit only runs in time mode.
- Session filter (last_ts ∈ nav window) still applied in Hud::ui for non-time mode.

## Problem/Context
Prior iteration had single nav bar whose meaning flipped by mode, creating circular deps: time window filtered sessions → shrunk max_turns → reactive turn domain. User wanted stable, fixed turn domain with independent time+turn scrolls.

## Solution/Approach
Two independent view states, two strips. Time strip never disappears. Turn strip is additive and mode-gated. Helper fn handles dots/viewport/drag/zoom/pan/labels uniformly.
