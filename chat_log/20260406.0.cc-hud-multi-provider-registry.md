## Problem/Context
User wants a "live spreadsheet" tracking energy/water/cost per model per time, with temporal table semantics for coefficient corrections. The current architecture is in-memory streaming with no persistence layer. The registry + OpenCode harness are the foundation for multi-provider tracking, but the temporal/SQLite persistence layer for cc-hud's own events hasn't been built yet.

## Solution/Approach
- Energy estimation for unknown models uses active parameter count as primary lever, benchmark-class equivalence as fallback, pricing bracket as last resort
- MoE models use active params not total (e.g. DeepSeek V3: 37B active of 671B total)
- Registry stores j_per_output_tok directly per model with EnergyTier fallback
- Confidence bands (+-3x) absorb estimation uncertainty
