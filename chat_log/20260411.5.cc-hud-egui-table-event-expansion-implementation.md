## Implementation Details

### New TableRow Variant
Added `SessionExpanded { si, gi, event_idx }` to the TableRow enum for displaying individual events.

### Event Cell Rendering
Implemented `render_event_cell()` method that displays:
- Column 2: Event type badge (color-coded), details, and secondary info line
- Color coding by event type:
  - API calls: Blue (100, 200, 255)
  - Tool uses: Orange (255, 200, 100)
  - Skill uses: Green (150, 255, 150)
  - File reads: Purple (200, 150, 255)
  - Agent spawns: Pink (255, 150, 200)
  - Compaction: Gray (150, 150, 150)

### Session Expansion Toggle
- Added `cell_expand_button()` in legacy module
- Shows ▶/▼ icon based on expansion state
- Click triggers toggle_session_expand action
- Column 3 now shows expand button instead of detail button

### Event Details Display
For each event type, displays:
- **ApiCall**: [API] + token counts (in/out), cost, model, elapsed time
- **ToolUse**: [Tool] + tool name
- **SkillUse**: [Skill] + skill name
- **ReadFile**: [File] + category
- **AgentSpawn**: [Agent] + type:description
- **Compaction**: [Compact] + context compaction

### Row Building Logic
Updated `LegendTableData::new()` to:
1. Add SessionExpanded rows for each event when session expanded
2. Show event rows before subagent rows
3. Maintain proper nesting structure

### Actions System
Added `toggle_session_expand: Vec<String>` to LegendActions for tracking session expansion toggles.

## Architecture Notes

### Row Height
Currently all rows use the same height (42.0) from `default_row_height()`. Event rows have extra vertical space which is used for:
- Type badge and details on first line
- Secondary info (model/time) on second line for API calls
- Cost aligned to right

### Column Layout
- Column 0: Eye (empty for events)
- Column 1: Swatch (empty for events)
- Column 2: Event type badge + details + secondary info
- Column 3: Expand button (empty for events)
- Column 4: Timeline (empty for events)
