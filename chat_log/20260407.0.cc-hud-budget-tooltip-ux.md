# Session: cc-hud-budget-tooltip-ux

## Goal
Fix visual bugs, add billing period budget chart, improve tooltip UX, and clean up chart rendering across cc-hud.

## Current State
All work committed across 2 commits on main (ahead of origin by 5):
- 8acfb3a: Budget chart, tooltip fixes, model labels, filter persistence
- 1fe4180: Limit input, bar cap, line smoothing, billing persistence

## Problem/Context
1. Cumulative lines zigzagged backwards in time mode (non-monotonic timestamps)
2. Tooltip showed stale data for sessions past their end, duplicate entries for same-project sessions
3. Pinned tooltip still followed mouse and updated data
4. No billing period cost tracking (usage chart going away for work accounts)
5. Model labels truncated (opus4 instead of opus4.6)
6. Include/exclude filter cleared on mode toggle
7. Bars absurdly wide when zoomed in on time axis
8. Step function rendering on cumulative lines looked wrong
9. $/Mtok rate in legend was confusing, user wanted raw cost + tokens
10. Tooltip row colors were all session-color, not matching bar segment colors

## Solution/Approach
### Bug fixes (0_scene.rs + main.rs)
- sort_by_x() on all line points before downsampling
- Skip sessions outside [first_x-5%, last_x+5%] in tooltip
- HashSet dedup on (name, detail) for tooltip entries
- Pinned tooltip stores HoverState + cursor pos, update_hover_src skips writes when pinned
- Sessions with duplicate project names indexed: sprefa(#1), sprefa(#2)
- nearby_skills deduped

### Budget chart (main.rs)
- BillingConfig: reset_day, limit_usd, web_reported total, persists to ~/.config/cc-hud/billing.json
- budget_cost_pts always uses time-based x (ts_x on TurnInfo), works in both axis modes
- Swaps with usage chart via "budget" toggle button in controls strip
- Web-reported input: text field for $ from dashboard, stamps current time, purple marker on chart
- Limit is text input, reset day is < > stepper
- In time mode links cursor with other charts; in turn mode auto-fits independently

### Rendering (0_scene.rs + main.rs)
- Bar width hard-capped at 5 minutes absolute
- Removed step_pts from all cumulative lines (only budget chart retains it)
- Model labels: opus4.6, opus4.5, opus4.1, sonnet4.6, etc.
- Include/exclude use independent sets (exclude_set, include_set)
- Removed $/Mtok from legend, show raw cost + tokens
- Tooltip rows use LayoutJob with per-column colors matching bar segments

## Key Insights
- step_pts only makes sense for budget chart (discrete spend jumps), not for cumulative overlay lines
- Bar width needs absolute cap, not proportional -- 0.2% of 36 days is still 100 minutes
- budget_cost_pts must be built from ts_x (always timestamps) to work in turn-index mode
- Include/exclude sets must be independent or toggling destroys selections
- Tooltip dedup needs HashSet not dedup_by -- sort key float differences scatter identical entries
