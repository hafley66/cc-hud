# Session: cc-hud-chart-cache-plotpoint

## Goal
Cache chart rendering data to eliminate per-frame allocations in egui immediate-mode Plot::show() calls.

## Current State
- ✅ Committed as 02f90b5: PlotCache stores pre-scaled+decimated line data as Vec<PlotPoint>
- ✅ Zero-copy rendering: pass &[PlotPoint] slices via PlotPoints::Borrowed (no per-frame Vec alloc for lines)
- ✅ PlotCache rebuilt only when (generation, hidden_set, time_axis) changes
- ⚠️ Bars still "broken" per user report - bars_culled reverted to original implementation but user says still dead
- ⚠️ Performance still not good enough - user says "perf is still bad"
- ⚠️ Need to verify user is running the new build (cargo run vs stale binary)

## Problem/Context
- egui immediate-mode: every frame rebuilds all Plot::show() elements from scratch
- With 6+ visible charts, each with bars/lines/points, frame time dominates
- Prior session (ed6abf7) added: bar viewport culling, line decimation, hover quantization, conditional repaint
- Those helped marginally but fundamental issue remained: ~80 Vec allocations per frame for line data

## Solution/Approach
Two-phase caching:
1. **PlotCache** struct (new, in main.rs) stores pre-computed line data as Vec<PlotPoint>
2. **Zero-copy rendering** via &[PlotPoint] -> PlotPoints::Borrowed (From<&'a [PlotPoint]>)

### What's cached (rebuilt only on data change):
- cost_lines, tok_in_lines, tok_out_lines, energy_lines, water_lines (per-session, scaled+decimated)
- totals_cost, totals_tok, totals_energy, totals_water (normalized+decimated)
- Scale factors (cost_scale, tok_scale, energy_scale, water_scale) baked into cached points

### What's still per-frame:
- bars_culled() x10 calls (builds Vec<Bar> from BarData each frame)
- Plot::show() x6+ (egui internal layout/transform/paint - the fundamental bottleneck)
- Hover alpha/width computation (cheap)
- Tooltip computation (iterates session turns)

## Key Insights
- egui_plot PlotPoints has Borrowed(&'a [PlotPoint]) variant via From<&'a [PlotPoint]> - zero alloc
- PlotPoint is {x: f64, y: f64}, From<[f64;2]> for PlotPoint exists
- Bar derives Clone, has pub fill field - can clone and mutate fill for hover highlighting
- Plot<'a> lifetime is covariant - Plot<'static> can be used where Plot<'shorter> needed, allowing closures that capture local refs
- Plot::show<'b> creates PlotUi<'b> internally, items with lifetime 'b added. Borrowed PlotPoints from cache outlive 'b since cache lives for entire draw_big call
- Bar pre-building attempt: build_bars + cull_bars (zip pre-built with BarData, filter+clone) was logically identical to bars_culled but user reported "broken in time mode". Root cause unknown - reverted.
- In time mode, show_bars defaults to false (toggled off when switching to time axis)
