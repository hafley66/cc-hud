# Session: cc-hud-tiles-grid-time-alpha

## Goal
Make all chart panes resizable via egui_tiles, and unfuck time mode which appeared blank.

## Current State
- Committed 4d56f72 (alpha fix) on top of b4b3ed2 (tiles full grid) on top of 8876fd1 (dual nav strips).
- egui_tiles 0.15 dep added; feature flag extended.
- All 7 chart panes (Cost, Tokens, Usage, Tool, Energy, Water, Totals) layout via egui_tiles Tree<ChartPane>. Tree persists across frames on Hud.
- Layout pattern: Behavior records ui.available_rect_before_wrap() per pane; existing chart render blocks draw into those recorded rects via ui.allocate_new_ui.
- gap_width=6.0, per-pane shrink(3.0) for inner margin.
- Time mode line alpha/width unified with turn mode (0.75/2.0). Time-toggle eprintln removed.
- Tiles demo window + right-top "tiles" toggle button remain (standalone proof).

## Problem/Context
- Prior dim defaults (is_time ? 0.30 : 0.75 alpha) made time-mode charts effectively invisible when few sessions were visible. User said "i see nothing".
- Root-caused via eprintln! trace showing hidden=246/249, time_view autofitting to fit range, data present but undrawn. Turned out to be alpha, not layout or state.
- User wanted all dashboard panes draggable-splitter, not just the bottom strip.

## Solution/Approach
- Introduce ChartPane enum + ChartBehavior that records rects during tree.ui() without drawing content.
- Replace all chart-row rect math (cost_rect, tok_rect, usage_rect, tool_rect, energy_wh_rect, water_ml_rect, totals_rect) with tree-assigned rects.
- Keep all existing chart rendering code unchanged; it renders into the tree-computed rects.
- Fix visibility regression by removing the is_time alpha dimming.

## Key Insights
- egui_tiles 0.15 matches egui 0.34; 0.14 pulls emath/epaint 0.33 and breaks Behavior trait signatures.
- Layout-only pattern (Behavior records rect, caller draws) is the cheapest way to integrate tiles without rewriting chart bodies or fighting lifetimes on a Behavior struct that borrows all rendering state.
- User-facing "blank" is often an alpha/scale issue, not a layout or state issue. Add debug eprintln before assuming state corruption.
- User prefers committing WIP over stashing before bisect/checkout ops (saved to memory as feedback_commit_before_stash).
