# Session: cc-hud-time-mode-bars

## Goal
Fix time mode rendering: bars overlapping/clustering, cumulative lines zigzagging, VLine marker noise.

## Current State

Time mode bars/lines are in their original (broken) state. Multiple fix attempts failed and made things worse.

## Problem/Context
In time mode with --history (1171 session files):
1. Bars from all sessions share x-axis (timestamp minutes). Multiple sessions at similar times produce overlapping bars at same x positions. Looks like "all bars in 1-2 places"
2. Per-session cumulative lines (N per visible session) interleave in time space creating visual noise/"zigzag"
3. Agent/skill VLine markers from all sessions create dense wall of vertical lines
4. egui_plot's stack_on() is INDEX-BASED (confirmed at egui_plot-0.29.0/src/items/mod.rs:1672) -- bars.get(index), no x-value matching. Stacked arrays MUST have same length with matching x at each index.

## Failed Approaches
1. Auto-disabling bars on time toggle -- user hated losing control
2. Reducing per-session line alpha in time mode -- didn't address root cause
3. Replacing per-session lines with combined cumulative line -- broke Y scaling (combined_cost_max >> total_cost_max)
4. Independent merge_bars() per array -- stacking misaligned because different arrays had different subsets of populated bins
5. merge_group() with aligned bins -- built correctly but still "ate shit", unclear why

## Key Insights
- egui_plot has NO histogram, NO auto-grouping, NO x-value-based stacking. All manual.
- stack_on stacks bar[i] on bar[i] by array index. Misaligned arrays = visual garbage.
- With 1171 sessions, per-session anything (bars, lines, markers) is inherently noisy in shared time space.
- The merge_group approach (all stacked arrays share same populated bin set, zero-height padding) is architecturally correct for the stacking problem. Something else is wrong with the visual output -- needs investigation with actual data.
- Time span with full history could be months, making fixed bin counts produce huge bin widths.
