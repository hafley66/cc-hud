# Session: remove-small-mode-and-fix-window-controls

## Goal
Remove "small mode" and make "big mode" always the default, plus create reusable dotted underline tooltip component for chart tooltips. Additionally, show agent harness name (e.g., "claude" vs "opencode") in session rows.

## Current State
✅ **COMPLETED**
- Small mode fully removed from codebase (1109 lines deleted)
- Big mode is now the only mode (no --big flag needed)
- History is now opt-out (--no-history flag) instead of opt-in
- Window uses OS-native decorations (title bar, resize handles, close button)
- Harness tags ([cc], [oc]) shown in session legend rows
- Context structs created to reduce function cardinality
- Header tooltips with dotted underlines implemented

## Problem/Context
User couldn't close window - it was pinned to the top. Discovered `set_floating(true)` and transparent window mode were interfering with mouse events and window controls.

## Solution/Approach
1. Removed all small mode code (draw_small, draw_strip, draw_tool_strip, pane_poll_loop)
2. Inverted history flag from opt-in --history to opt-out --no-history
3. Initially added custom egui window controls (title bar, close button, resize handle)
4. User feedback: wanted OS-native controls instead
5. Switched to OS-native decorations by removing set_decorated(false)
6. Fixed uncloseable window by removing:
   - set_floating(true)
   - NSWindowCollectionBehavior flags
   - transparent_window: true (changed to false)
