# Session: cc-hud-perf-filters-ux

## Goal
Fix performance choking on large datasets (459M tokens, 100+ sessions, 36+ days), add include/exclude filtering, restore active filter, reduce visual noise.

## Current State
All requested work committed across 3 commits:
- 5a09585: Perf fixes, include/exclude filter, tooltip pin in all modes
- f271bfa: Active filter, bars toggle, bar width cap
- Prior session (04e38b3): Multi-provider model registry, OpenCode harness

## Problem/Context
1. egui_plot choking on 500k+ Bar objects per frame (100 sessions * 400 bars * 13 arrays)
2. Cumulative line series unbounded (every API call = one point per session)
3. build_chart_data rebuilt every frame even when data unchanged
4. Bar width scaling linearly with time span, producing absurdly fat bars over 36+ days
5. No way to filter to specific sessions or hide bars to see only cumulative lines
6. Old show_active_only was removed when include/exclude was added, but user wanted both

## Solution/Approach
### Performance (0_scene.rs)
- Bar downsampling now globally bounded: max_bars_per_session = 600 / visible_sessions
- All cumulative line series downsampled to 300 points via uniform sampling (downsample_line())
- Bar width capped at 0.5% of total time span (.min(time_span_min * 0.005))
- generation counter on HudData, chart data cached across frames in Hud.cached_chart

### Filtering (main.rs)
- FilterMode enum (Include/Exclude) with filter_set: HashSet<String>
- Include mode: only sessions in set are shown; Exclude mode: sessions in set are hidden
- show_active_only restored as independent toggle stacking with include/exclude
- show_bars toggle hides per-turn bars, showing only cumulative lines
- Toggle buttons in controls strip: include/exclude, active, bars, time, fit, plus chart vis

### UX
- Tooltip click-to-pin works in both turn and time axis modes (was gated behind is_time)
- Stats text brightness increased (dim_col from rgba(130,120,100,alpha/2) to rgba(170,160,140,alpha*3/4))
- Stats use monospace fixed-width format strings for cross-row alignment

## Key Insights
- egui_plot Bar objects are the main bottleneck, not line points -- each Bar is a separate draw call
- Downsampling must account for session count multiplicatively (N sessions * M bars = N*M total)
- generation counter on shared data is the cheapest way to avoid per-frame rebuilds
- include/exclude is the right mental model for filtering -- exclude hides specific items, include shows only specific items, and active is orthogonal to both
- Bar width must be capped as a fraction of span, not computed from bucket size, because bucket size grows with span/session ratio
