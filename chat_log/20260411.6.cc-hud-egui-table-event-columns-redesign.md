## Problem/Context
- Initial implementation used SessionExpanded rows causing visual bugs (flashing thick red rectangles)
- User clarification: wanted event states as **columns** not expanded rows
- Becomes unresponsive on startup (possibly due to plot rendering)
- "Super weird" visual appearance with expansion design

## Solution/Approach
1. Removed SessionExpanded row type and render_event_cell method
2. Added event type columns to table layout (4 new columns between session and detail)
3. Aggregate event counts per session in render_session_cell using fold
4. Updated headers: ["", "", "Session", "API", "Tools", "Skills", "Files", "", "Timeline"]
5. Simplified actions system (removed toggle_session_expand)
6. Set show_active_only default to true for better initial state
