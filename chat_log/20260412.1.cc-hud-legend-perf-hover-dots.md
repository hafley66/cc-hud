## Problem/Context
- egui is immediate-mode: every frame calls Plot::show() which rebuilds all bars/lines/points from scratch
- With 6+ charts visible, each containing hundreds of bars and lines, frame time is too high
- Hover triggers continuous repaint (egui requests repaint on any pointer movement)
- Non-time mode especially slow because all bars are "in viewport" (turn indices overlap)
- Three haiku agents were spawned for parallel work but their worktree changes had issues (wrong API calls, wrong file locations). Manually applied and fixed all changes.

## Solution/Approach
Perf optimizations applied (incremental, not sufficient for full fix):
1. bars_culled() - viewport x-range filtering before creating Bar elements
2. decimate() - stride-based line point reduction to 400 max
3. Conditional pui.points() - skip dot markers when dense
4. Hover x quantization - round to hl_threshold steps so hovered_sessions only changes at bucket boundaries
5. Conditional repaint - generation-based immediate + 250ms polling + interaction-only

## Key Insights
- egui_table 0.8.0 Column::auto_size_this_frame(bool) exists on Column, NOT on Table. Setting it causes request_discard (frame re-render).
- TableState::reset(ctx, id) clears persisted column widths, forcing full re-layout. But it FIGHTS with AutoSizeMode::OnParentResize causing flicker. Removed debounce entirely.
- AutoSizeMode::OnParentResize already handles resize correctly: checks state.parent_width != Some(parent_width), calls Column::auto_size, stores new width. No debounce needed.
- LegendHighlight stores (f64, f64) time ranges in minutes-from-epoch. Charts read via egui temp data. Only draw VLines in time mode (x-axis is minutes-from-epoch; non-time mode uses turn index).
- Spawning 3 haiku agents in parallel with worktree isolation: agents made errors (wrong API methods like auto_size_this_frame on Table, modifying wrong request_repaint location). Had to manually review and fix all agent output. Agents useful for parallelism but need very precise instructions about API availability.
- `gen` is a reserved keyword in Rust 2024 edition - use `data_gen` instead
- Per-project session index: pre-compute by sorting sessions per project by first_ts ascending, store in HashMap<usize, usize> (session_index -> rank). Both legend table and chart tooltips use same indexing.
- Hover quantization: round effective_hover_x to hl_threshold grid, so hovered_sessions Vec<bool> only changes when cursor crosses session boundaries, not every pixel.
- bars_to_egui_hl replaced by bars_culled which adds viewport filtering. Dead code removed.
